# HEART对话演练系统 - 使用说明

## 快速开始

### 1. 启动服务

```bash
# 启动后端（终端1）
cd backend
node index.js

# 启动前端（终端2）
cd frontend
npm start
```

访问：http://localhost:3000

### 2. 使用流程

#### 第一步：输入场景
在首页输入你想练习的沟通场景，例如：
```
我是团队负责人，需要和下属小李沟通他最近的工作表现。
他是资深工程师，最近两个月交付延期，代码质量下降。
我希望了解原因并帮助他改进。
```

#### 第二步：HEART五步演练
系统会引导你完成五个步骤，每步都有通过标准：

**H步 - 倾听（20字+关键词）**
- 必须包含：听到、了解、你说的是、你的意思是、我理解你说
- 示例："我听到你说工作压力很大，需求变动频繁"

**E步 - 共情（15字+关键词）**
- 必须包含：理解、感受、能想象、可以理解、换位思考
- 示例："我能理解你的感受，同时处理多个模块确实压力很大"

**A步 - 提问（10字+疑问句）**
- 必须包含：?、吗、呢、如何、怎么、为什么
- 示例："你觉得哪些模块最影响进度？有什么具体困难吗？"

**R步 - 回应（20字+行动词）**
- 必须包含：建议、可以、我们、方案、计划、行动
- 示例："我建议每周安排需求评审会，技术选型我可以安排架构师协助"

**T步 - 追踪（15字+时间词）**
- 必须包含：时间、截止、检查、跟进、确认、下次
- 示例："下周三开始，每周三下午3点。两周后我们再检查效果"

#### 第三步：查看复盘
完成五步后自动跳转复盘页面，查看：
- HEART五项评分（0-100分）
- 亮点总结
- 改进建议
- 行动计划
- 可导出为图片

## 使用技巧

1. **善用可点用话术**：右侧蓝色文字可直接点击填入输入框
2. **注意教练提示**：黄色区域会给出实时建议
3. **关注通过标准**：每步都有明确的关键词和字数要求
4. **多次尝试**：未通过可继续对话，直到满足条件

## 常见问题

**Q: 为什么一直停在某一步？**
A: 检查是否包含该步骤的关键词，可点击右侧话术参考

**Q: 可以跳过某一步吗？**
A: 不可以，必须按H-E-A-R-T顺序完成

**Q: 对话可以进行多少轮？**
A: 每步不限轮数，满足条件即可进入下一步

**Q: 如何重新开始？**
A: 刷新页面或返回首页重新输入场景

## 停止服务

```bash
pkill -f "node index.js"
pkill -f "react-scripts"
```
